# BanJiuOS 操作系统需求文档

## 1. 项目概述

BanJiuOS 是一个现代开源操作系统，采用 EFI 引导，具有自定义的可执行文件格式，提供图形化桌面环境和命令行界面。

## 2. 目标架构

- **目标平台**: x86_64 (AMD64)
- **最小硬件要求**:
  - CPU: Intel/AMD x86_64 处理器
  - 内存: 4GB RAM
  - 存储: 32GB 磁盘空间
  - 显卡: 支持 UEFI GOP (Graphics Output Protocol)

## 3. 文件系统方案

| 分区 | 文件系统 | 用途 | 大小建议 |
|------|----------|------|----------|
| EFI System Partition | FAT32 | 引导文件存放 | 512MB |
| 系统分区 | EXT4 | 操作系统文件 | 20GB |
| 用户分区 | NTFS | 用户数据（兼容Windows） | 剩余空间 |

## 4. EFI 引导程序需求

### 4.1 引导流程
1. UEFI BIOS 加载 BanJiuOS.efi
2. 初始化硬件检测
3. 加载内核到内存
4. 切换到保护模式/长模式
5. 跳转到内核入口

### 4.2 引导程序功能
- 支持 UEFI 2.3.1+ 规范
- 支持 GPT 分区表
- 支持 FAT32/EXT4 文件系统读取
- 显示启动画面（Logo + 进度条）
- 提供启动菜单（多内核选择、安全模式）

### 4.3 EFI 引导程序文件结构

| 文件路径 | 说明 | UEFI要求 |
|----------|------|----------|
| `EFI/BOOT/BOOTX64.EFI` | 主引导程序 (x86_64) | **必须**，BIOS默认查找 |
| `EFI/BOOT/BOOTIA32.EFI` | 32位引导程序 | 可选 |
| `EFI/BOOT/BANJULDR.EFI` | 自定义加载器 | 可选 |

### 4.4 ISO 镜像结构

```
BanJiuOS-ISO/
├── EFI/
│   └── BOOT/
│       ├── BOOTX64.EFI      # x86_64 主引导程序
│       ├── BOOTIA32.EFI      # i386 备用引导程序
│       └── Fonts/
│           └── unicode.pf2   # UEFI字体
├── Boot/
│   ├── grub.cfg             # GRUB配置（或自定义配置）
│   └── themes/               # 启动主题
├── System/

│   ├── initramfs.img        # 初始文件系统
│   └── loader.zyxt          # 系统加载器
├── Installer/
│   ├── setup.exe            # 安装程序
│   └── setup.zyxt           # 安装程序（BJOS格式）
└── Media/
    └── ...                   # 安装介质资源
```

## 5. 安装程序规范

### 5.1 安装程序功能

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 磁盘分区 | 创建/删除/调整 GPT 分区 | 必须 |
| 文件系统格式化 | EXT4/FAT32/NTFS 分区格式化 | 必须 |
| 系统文件复制 | 复制系统文件到目标分区 | 必须 |
| 引导程序安装 | 安装 EFI 引导程序到 ESP | 必须 |
| 用户设置 | 设置用户名/密码/时区 | 可选 |

### 5.2 安装流程

1. **启动安装程序** - 从 EFI/BOOT/BOOTX64.EFI 启动
2. **选择安装目标磁盘** - 列出可用磁盘供用户选择
3. **磁盘分区** - 自动或手动分区
4. **格式化分区** - 选择文件系统类型
5. **选择要安装的组件** - 内核、驱动、工具
6. **复制系统文件** - 将系统文件复制到目标分区
7. **安装引导程序** - 配置 EFI 引导项
8. **完成安装** - 提示重启

### 5.3 分区方案

| 分区 | 大小 | 文件系统 | 挂载点 | 说明 |
|------|------|----------|--------|------|
| ESP | 512MB | FAT32 | /boot/efi | EFI系统分区 |
| Root | 20GB+ | EXT4 | / | 根分区 |
| Home | 剩余 | NTFS | /home | 用户数据 |

### 5.4 安装程序参数

| 参数 | 说明 |
|------|------|
| `--disk=/dev/sda` | 指定安装目标磁盘 |
| `--partition` | 自动分区 |
| `--no-efi` | 跳过EFI引导安装 |
| `--username=<name>` | 设置管理员用户名 |
| `--password=<pass>` | 设置管理员密码 |
| `--verbose` | 详细输出 |

### 5.5 引导程序安装

安装程序需要：
1. 创建 EFI System Partition (ESP)
2. 复制 BOOTX64.EFI 到 ESP:/EFI/BanJiuOS/
3. 创建 NVRAM 启动项指向 BanJiuOS 引导程序
4. 配置引导菜单项

## 6. 自定义可执行文件格式

### 5.1 文件魔数

BanJiuOS 可执行文件格式以 **"BJOS"** 的 UTF-8 编码作为定界符，用于标识文件的开头和结尾。

- **开头魔数**: `0x42 0x4A 0x4F 0x53` ("BJOS" UTF-8)
- **结尾魔数**: `0x42 0x4A 0x4F 0x53` ("BJOS" UTF-8)

### 5.2 文件结构

```
+------------------+
|  Magic (4B)     |  开头魔数 "BJOS" (0x42 0x4A 0x4F 0x53)
+------------------+
|  Header Size (4) |  文件头大小
+------------------+
|  Format Ver (2) |  格式版本号 (主.次)
+------------------+
|  File Size (8)   |  总文件大小 (字节)
+------------------+
|  Entry (8)       |  入口点虚拟地址
+------------------+
|  Section Count(4)|  节表项数量
+------------------+
|  Resource Off(8) |  资源段文件偏移
+------------------+
|  Flags (4)       |  标志位
+------------------+  <-- Header Size 结束
|                  |
|  Section Table    |  节表 (Section Count × 56字节)
|                  |
+------------------+
|                  |
|  Sections Data   |  各节数据区
|                  |
+------------------+
|                  |
|  Resource Section|  资源段 (图标/字符串/位图)
|                  |
+------------------+
|  Magic (4B)     |  结尾魔数 "BJOS" (0x42 0x4A 0x4F 0x53)
+------------------+
```

### 5.3 文件头结构

| 字段 | 大小 | 说明 |
|------|------|------|
| Magic | 4字节 | 文件开头魔数 "BJOS" |
| HeaderSize | 4字节 | 文件头大小（不含魔数） |
| FormatVersion | 2字节 | 格式版本（高字节主版本，低字节次版本） |
| FileSize | 8字节 | 总文件大小（含开头/结尾魔数） |
| EntryPoint | 8字节 | 入口点虚拟地址 |
| SectionCount | 4字节 | 节表项数量 |
| ResourceOffset | 8字节 | 资源段文件偏移（相对于文件起始） |
| Flags | 4字节 | 标志位（bit0=PRIVILEGED, bit1-31=系统保留） |

### 5.4 节表结构

| 字段 | 大小 | 说明 |
|------|------|------|
| Name | 16字节 | 节名称（NULL填充） |
| Type | 4字节 | 节类型 |
| Offset | 8字节 | 文件偏移（相对于文件起始） |
| Size | 8字节 | 节大小（内存中大小） |
| VirtualAddress | 8字节 | 虚拟地址 |
| Flags | 4字节 | 节标志 |

### 5.5 节类型定义

| 类型值 | 名称 | 说明 |
|--------|------|------|
| 0x00000001 | CODE | 可执行代码 |
| 0x00000002 | DATA | 初始化数据 |
| 0x00000004 | BSS | 未初始化数据 |
| 0x00000008 | RODATA | 只读数据 |
| 0x00000010 | ICON | 图标资源 |
| 0x00000020 | STRING | 字符串资源 |
| 0x00000040 | BITMAP | 位图资源 |
| 0x00000080 | CUSTOM | 自定义资源 |
| 0x80000000 | PRIVILEGED | 需要 Ring 0 权限运行 |

### 5.6 资源段结构

#### 5.6.1 图标段 (ICON)
- 支持多种尺寸: 16x16, 32x32, 64x64, 128x128, 256x256
- 支持 PNG 格式压缩
- 图标ID + 尺寸 + 数据偏移 + 数据大小

#### 5.6.2 字符串段 (STRING)
- 支持多语言（UTF-16编码）
- 字符串ID + 语言代码 + 字符串内容

#### 5.6.3 位图段 (BITMAP)
- 支持 BMP/PNG 格式
- 位图ID + 宽度 + 高度 + 位深度 + 数据

## 9. 安全特性

### 9.1 权限模型
- **默认权限**: 所有程序（内核程序和用户程序）默认以 **Ring 3 (用户态)** 权限运行
- **Ring 0 声明机制**: 程序编译时可声明需要 Ring 0 权限，操作系统 **无需校验即可直接以 Ring 0 运行**
- **设计原则**: 信任开发者，简化权限校验流程，提高系统运行效率

### 9.2 权限标志位
| 标志 | 名称 | 说明 |
|------|------|------|
| 0x00000001 | PRIVILEGED | 声明需要 Ring 0 权限运行 |

### 9.3 文件头 Flags 字段扩展
- **低 8 位**: 保留给权限标志
- **高 24 位**: 系统保留

### 9.4 用户权限管理
- UID/GID 用户身份标识
- 文件权限（读/写/执行）
- 进程权限继承

### 9.5 内存保护
- NX位（No-Execute）保护
- ASLR（地址空间布局随机化）
- 内核态/用户态内存隔离

### 9.6 可选安全模块
- SELinux 风格强制访问控制（MAC）
- Audit 审计日志

## 10. 链接器规范

### 10.1 文件类型

| 文件类型 | 后缀 | 说明 |
|----------|------|------|
| 可执行文件 | `.zyxt` | BanJiuOS 可执行文件格式 |
| 目标文件 | `.o` | 可重定位目标文件 |
| 静态库 | `.a` | 归档的目标文件集合 |

### 10.2 链接器参数

| 参数 | 说明 |
|------|------|
| `-o <file>` | 指定输出文件名（默认：`a.zyxt`） |
| `-static` | 全静态链接（默认，除了内核模块） |
| `-dynamic` | 启用动态链接 |
| `-l<name>` | 链接库文件 `lib<name>.a` |
| `-L<dir>` | 添加库搜索路径 |
| `-e <addr>` | 指定入口点地址 |
| `-T <script>` | 使用链接脚本 |
| `-Map=<file>` | 生成链接映射文件 |
| `--verbose` | 详细输出模式 |
| `-S` | 去除调试信息 |
| `-s` | 去除所有符号信息 |

### 10.3 链接器默认行为

- **默认链接模式**: 全静态链接（内核模块除外）
- **内核模块链接**: 内核模块始终动态链接，由内核加载器负责解析符号
- **输出文件名**: 未指定时默认为 `a.zyxt`
- **入口点**: 未指定时使用符号 `_start` 或首节入口地址

### 10.4 BJOS 目标文件格式 (.o)

BJOS 目标文件与可执行文件格式相似，但有以下区别：

| 特性 | 可执行文件 (.zyxt) | 目标文件 (.o) |
|------|-------------------|---------------|
| 开头魔数 | BJOS | BJOS |
| 结尾魔数 | BJOS | 无 |
| FileSize | 总大小 | 0（由链接器计算） |
| EntryPoint | 程序入口 | 0（待链接确定） |
| Flags | 运行时标志 | 0x00000001 (可重定位) |
| Sections | 程序节 | 代码/数据节 |

### 10.5 目标文件头结构 (.o)

```
+------------------+
|  Magic (4B)     |  开头魔数 "BJOS" (0x42 0x4A 0x4F 0x53)
+------------------+
|  Header Size (4) |  文件头大小
+------------------+
|  Format Ver (2) |  格式版本号 (主.次)
+------------------+
|  File Size (8)   |  0（链接时计算）
+------------------+
|  Entry (8)       |  0（待链接确定）
+------------------+
|  Section Count(4)|  节表项数量
+------------------+
|  Resource Off(8) |  资源段偏移（无则为0）
+------------------+
|  Flags (4)       |  0x00000001 (OBJ_REL)
+------------------+
|  Symbol Count(4) |  符号数量（新增）
+------------------+
|  Reloc Count (4) |  重定位条目数量（新增）
+------------------+  <-- Header Size 结束
|                  |
|  Section Table    |  节表
+------------------+
|                  |
|  Sections Data   |  各节数据区
+------------------+
|                  |
|  Symbol Table    |  符号表
+------------------+
|                  |
|  Relocation Table|  重定位表
+------------------+
```

### 10.6 符号表结构

| 字段 | 大小 | 说明 |
|------|------|------|
| Name | 64字节 | 符号名称（NULL填充） |

### 10.8 重定位类型

| 类型值 | 名称 | 说明 |
|--------|------|------|
| 0x01 | R_ZYXT_64 | 64位绝对地址 |
| 0x02 | R_ZYXT_32 | 32位绝对地址 |
| 0x03 | R_ZYXT_PC | PC相对地址 |
| 0x04 | R_ZYXT_16 | 16位绝对地址 |

## 附录: Ring0/ Ring3 权限模型说明

### BanJiuOS 权限模型
- **默认行为**: 所有程序以 Ring 3 用户态运行
- **Ring0 声明**: 程序通过设置 PRIVILEGED 标志（0x80000000）声明需要 Ring 0
- **执行策略**: 声明 Ring0 的程序无需校验，直接以 Ring 0 运行
- **设计理念**: 信任开发者编写的程序，简化权限检查，提升性能

### 对比 Linux/Windows
| 系统 | 权限模型 | Ring0 程序 |
|------|----------|-------------|
| BanJiuOS | 默认Ring3，编译声明Ring0 | 声明即运行 |
| Linux | 普通用户态，内核态分离 | 需要系统调用 |
| Windows | 用户态/内核态分离 | 需要驱动签名 |
